# PRD: 나스닥 종목추천 (MVP v2)

## 1\. 개요

|항목|내용|
|-|-|
|서비스명|나스닥 종목추천|
|한 줄 정의|기술적 지표 기반으로 나스닥100 종목 중 유망 종목을 추천하고, 위험 성향별 포트폴리오와 예상 수익률을 보여주는 웹 서비스|
|목적|투자 초보자가 복잡한 분석 없이도 "지금 살만한 종목"에 대한 힌트를 얻을 수 있게 한다|
|타겟 유저|미국 주식(나스닥)에 관심 있는 개인 투자자, 특히 기술적 분석에 익숙하지 않은 초보자|
|개발 난이도 목표|첫 바이브코딩 프로젝트 — 화면 4개, 백엔드 없는 구조 유지. 단, v1 대비 검색 필터·포트폴리오 로직의 계산 복잡도는 다소 상승|

## 2\. 배경 및 문제 정의

* 개인 투자자는 나스닥 100개 이상 종목 중 어떤 것을 볼지부터 막막함을 느낀다.
* 이동평균선, 거래량, RSI 같은 기술적 지표는 존재하지만 이를 한눈에 종합해서 보여주는 도구가 부족하다.
* 원래 아이디어(13F 기관투자 분석, S\&P500 포함, 멀티 에이전트 운영 등)는 완성도는 높지만 데이터 소스 확보와 개발 난이도가 높아 1차 버전에서는 제외한다.

## 3\. MVP 범위 (MoSCoW)

### Must have

1. 종목 검색 (상세 필터 포함): 나스닥100 종목 리스트에서 이름/티커 검색 + 아래 4개 조건으로 필터링

   * 이동평균선 이격도 (현재가와 20일 이평선의 괴리율)
   * 거래량 증가 여부 (최근 5일 평균 거래량 vs 이전 20일 평균)
   * 주도주 섹터 (최근 3개월 섹터별 평균 수익률 상위 섹터)
   * 과열/과매도 상태 (RSI 14일 기준, 70 이상 과열 / 30 이하 과매도)
2. 종목 추천: RSI≥50·MACD 골든크로스로 매수 신호를 판정한 뒤, 이평선 이격도·거래량·주도섹터로 순위를 매기는 2단계 추세추종 로직으로 상위 5\~10개 종목 추천
3. 추천 이유 표시: 지표 기반 한 줄 설명
4. 과거 3개월 시뮬레이션: 추천 종목을 3개월 전에 샀다면 얻었을 예상 수익률 계산
5. 위험 성향별 포트폴리오 구성: 안정형 / 균형형 / 공격형 3가지 유형으로 종목 비중을 다르게 구성해 각각의 예상 수익률 제시
6. 선택 상태 유지: 새로고침해도 검색 조건·선택한 종목이 유지됨 (브라우저 localStorage 활용)

### Should have

7. 리밸런싱 시점 알림 (단순 규칙 기반, 예: "20% 이상 등락 시 검토 권장")

### Could have

8. 손절/익절 자동 전략 제안
9. S\&P500까지 종목 범위 확대
10. 13F 보고서 기반 기관투자자 매수 분석

### Won't have

* 멀티 에이전트 구조 (투자관리 에이전트 + 서브에이전트 체계)
* 실시간 자동매매 및 실제 증권사 계좌 연동 (매매 없음, 데이터 조회만)
* 로그인/회원 시스템 (익명 사용, 브라우저 로컬 저장으로 대체)
* 실시간 시세 갱신 (사전 수집한 데이터 기준, 실시간 아님)

## 4\. 핵심 기능 상세

### 4.1 종목 검색 (상세 필터)

* 기본: 나스닥100 종목 대상 티커/종목명 검색, 섹터 필터
* 상세 조건 (체크박스 또는 슬라이더로 조합 가능):

  * 이평선 이격도: (현재가 - 20일 이동평균) / 20일 이동평균 x 100 (%) — 예: "+5% 이상"만 보기
  * 거래량 증가: 최근 5일 평균 거래량이 이전 20일 평균 대비 일정 % 이상 증가한 종목만 필터
  * 주도주 섹터: 최근 3개월 섹터별 평균 수익률을 계산해 상위 N개 섹터에 "주도 섹터" 태그 부여, 해당 섹터만 필터
  * 과열/과매도: RSI(14일) 계산 후 70 이상(과열) / 30 이하(과매도) 상태로 필터
  * 위 지표들은 모두 사전 수집한 가격·거래량 데이터로 클라이언트에서 직접 계산 (별도 서버 계산 불필요)

### 4.2 종목 추천 (추세추종 신호 판정 + 순위 스코어링)

일반적인 추세추종 전략 규칙에 따라 "매수 신호 판정"과 "순위 점수"를 2단계로 분리한다.

**1단계 — 매수 신호 판정 (모두 충족해야 통과)**

|조건|기준|
|-|-|
|RSI(14)|50 이상|
|MACD|MACD선이 0선 위에 있음 (12일 EMA > 26일 EMA)|
|골든크로스|최근 5거래일 이내 MACD선이 시그널선(MACD의 9일 EMA)을 상향 돌파|

세 조건을 모두 만족하는 종목만 "추세 상승" 종목으로 분류해 2단계 순위 계산 대상에 포함한다.

**2단계 — 순위 점수 (1단계 통과 종목 대상, 100점 만점)**

|요소|배점|계산 기준|
|-|-|-|
|이평선 이격도|0\~60|(현재가-20일 이평)/20일 이평 x 100 값이 클수록 높은 점수 (상한 있음)|
|거래량 증가|0\~30|최근 5일 평균 거래량 증가율이 클수록 높은 점수 (상한 있음)|
|주도섹터 가산점|0\~10|종목이 속한 섹터가 "주도 섹터"(4.1 기준)에 해당하면 10점, 아니면 0점|

* 최종 순위: 1단계를 통과한 종목을 2단계 점수 순으로 정렬해 상위 5\~10개 추천
* **예외 처리**: 1단계 통과 종목이 5개 미만이면 조건을 단계적으로 완화한다 → ① 골든크로스 기준을 5일→10일로 확대 → ② 그래도 부족하면 MACD 조건은 유지하되 RSI 기준만 적용 → 최소 5개는 채운다 (완화된 경우 화면에 "조건 완화 적용됨" 표시)
* 결과 화면: 종목 리스트 + 순위 점수 + 추천 이유 한 줄 (예: "RSI 62, MACD 골든크로스, 주도 섹터 소속")

### 4.3 시뮬레이션 \& 예상 수익률

* 최근 3개월 가격 데이터 기준 "3개월 전 매수했다면"의 결과를 계산 (실제 미래 예측이 아님을 화면에 명확히 고지: "과거 3개월 실현 수익률 (미래 보장 아님)")
* 수익률(%), 최고가/최저가 대비 변동폭 표시

### 4.4 위험 성향별 포트폴리오 구성

* 추천 종목 중 사용자가 체크박스로 3\~5개 선택
* 변동성 계산: 각 종목의 최근 3개월 일별 수익률 표준편차로 변동성 산출
* 3가지 포트폴리오 유형:

  * 안정형: 변동성이 낮은 종목에 높은 비중 → 예상 수익률 낮지만 변동폭 작음
  * 균형형: 균등 비중 → 중간 위험·수익
  * 공격형: 변동성이 높은 종목에 높은 비중 → 예상 수익률 높지만 변동폭 큼
* 3개 유형 각각의 예상 수익률과 변동폭을 나란히 비교 표시

### 4.5 선택 상태 유지 (신규)

* 검색 조건, 선택한 종목, 마지막으로 본 포트폴리오 유형을 브라우저 localStorage에 저장
* 새로고침 시 저장된 상태를 불러와 화면에 복원
* 계정/로그인 없이 "이 브라우저에서만" 유지되는 방식 (기기 변경 시 초기화됨을 화면에 안내)

## 5\. 화면 구성 (4개 화면, SPA 구조)

|화면|설명|이전 화면에서 전달받는 상태|
|-|-|-|
|1. 홈/검색 화면|종목 검색 + 상세 필터(4종)|localStorage에서 복원된 이전 조건|
|2. 추천 결과 화면|추천 종목 리스트 + 추천 이유 + 체크박스 선택|1번의 검색/필터 조건|
|3. 시뮬레이션/수익률 화면|선택 종목의 3개월 시뮬레이션 결과|2번에서 선택한 종목|
|4. 포트폴리오 화면|안정형/균형형/공격형 3유형 비교|2번에서 선택한 종목, 3번의 시뮬레이션 결과|

4개 화면은 별도 페이지 이동 없이 하나의 앱 안에서 전환되며(SPA), 화면 간 데이터는 최상위 컴포넌트가 갖고 있다가 각 화면에 전달하는 구조로 통일한다 (6장 기술 스택 참고).

## 7\. 데이터 소스 (확정)

* 방식: API로 실제 데이터를 가져오되, 서비스 실행 시에는 백엔드 없이 정적 파일만 사용
* 수집 절차:

  1. 한국투자증권 KIS Open API(또는 대체 무료 API)로 나스닥100 종목의 최근 3개월 일별 가격·거래량 데이터를 로컬에서 1회 수집하는 별도 스크립트 작성 (Python, requests/pandas 활용)
  2. 수집 결과를 JSON 파일로 저장해 repo에 포함 (예: data/nasdaq100\_3m.json)
  3. 웹앱은 이 JSON 파일만 읽어서 검색·추천·시뮬레이션·포트폴리오 계산에 사용
* 장점: 실제 데이터 기반이면서도 API 키가 브라우저에 노출되지 않고, 실행 중 백엔드 서버가 필요 없음
* 한계: 실시간 갱신은 되지 않으며, 데이터를 다시 최신화하려면 수집 스크립트를 재실행해야 함 (이번 범위에서는 문제 없음)

## 8\. 비기능 요구사항

* 대상 종목 범위: 나스닥100 (S\&P500은 다음 버전에서 확대)
* 성능: 지표 계산(이격도, RSI 등)은 모두 클라이언트에서 정적 JSON 기반으로 처리, API 실시간 호출 없음
* 반응형: 데스크톱 기준 우선 개발, 모바일은 필수 아님
* 면책 문구: 모든 추천/시뮬레이션/포트폴리오 화면에 "투자 참고용이며 투자 손실 책임은 본인에게 있음" 고지 필수

## 9\. 성공 기준 (이번 과제 기준)

* \[ ] 종목 검색(상세 필터 포함)부터 추천, 시뮬레이션, 포트폴리오 구성까지 4개 화면이 끊김 없이 연결된다
* \[ ] 검색 상세 필터 4종(이격도/거래량증가/주도섹터/과열·과매도)이 실제 데이터 기준으로 동작한다
* \[ ] 안정형/균형형/공격형 3가지 포트폴리오 유형의 비중과 예상 수익률이 서로 다르게 계산된다
* \[ ] 새로고침 후에도 검색 조건과 선택 종목이 유지된다
* \[ ] 첫 개발 과제로서 로컬 및 GitHub Pages 배포 환경에서 데모 가능한 수준으로 완성된다

## 10\. 향후 확장 아이디어 (이번 범위 아님, 참고용)

* 투자관리 에이전트 + 서브에이전트(종목검색추천/포트폴리오구성/포트폴리오운영) 체계로 고도화
* 13F 보고서 기반 기관투자자 매수 동향 분석
* 손절/익절 자동 전략 및 리밸런싱 알림 고도화
* S\&P500까지 종목 유니버스 확대
* 실시간 데이터 갱신 (백엔드 도입 시점에 함께 검토)
