## 프로젝트 컨텍스트 (매 이터레이션 필독)

- v9.1 자산 위에서 작업. 추천·백테스트·변형·신선도 인프라 전부 가동 중
- 절대 원칙 (기존 + v10 확장):
  - **같은 코드 원칙 확장**: 국면 판정·진입 상태·청산 신호도 `src/lib/*`에 구현하고
    백테스트는 그것을 import한다. scripts/ 쪽 재구현 금지 (변형의 "정책 wrapper"만 scripts/ 허용)
  - **미래 참조 불가**: 국면·피벗·청산 신호 전부 슬라이스된 데이터로만 판정
  - **자동 최적화 금지 / adopted 항상 false / 채택·상수 변경은 운영자 몫**
  - **Out-of-Sample만 화면 표시** / 신규 파일(ngx100.json, fundamentals_ngx.json,
    backtest_ngx.json, backtest.json v3 필드) 부재 시 해당 기능만 비표시 (graceful degradation)
  - **기존 동작 변경 금지**: 추천 로직·기존 화면 요소는 그대로. v10은 추가만
  - **검증 상태 정직 표기**: UI에 노출되는 진입·청산 규칙은 "검증 우위/검증 열위/측정 중"
    상태를 반드시 병기. −8% 손절은 "단일 상승 국면 검증 열위"로 표기 (권장처럼 보이면 실패)
  - 새 npm 의존성 금지, 기존 테스트 전체 통과 유지
- 신규 상수는 `src/lib/constants/{regime,entry,exit}.js` (changelog 규약 공통,
  전부 "설계값 — 백테스트/국면 분해로 사후 검증" 주석)
- 품질 게이트 (모든 스토리 공통): `npm run test` 전체 통과 + `npm run build` 성공

### US-2. 국면 지표 lib
- [x] **개발자로서, 시장 폭 기반 3상태 국면을 결정적으로 판정하는 공용 함수를 원한다.**

**구현:** `src/lib/regime.js` + `constants/regime.js`
- `breadth(universe)` = dataSufficient 종목 중 close > SMA200 비율 (기존 SMA 재사용)
- `regimeSeries(universe)` = 일자별 국면 시계열 — **히스테리시스** 전이 규칙 (PRD §4.2 상수:
  상승↔중립 0.65/0.55, 중립↔하락 0.50/0.40), 초기 상태 규칙 포함
- `currentRegime(universe)` = 최신 시점 국면 + breadth 값 + 직전 전환일
- 순수 함수, 슬라이스된 유니버스 입력 전제 (미래 참조 없음)

**승인 기준:**
1. 히스테리시스 테스트: 경계 진동 시계열에서 상태가 깜빡이지 않음 (단일 문턱이면 실패하는 케이스) — 4개 이상
2. 초기 상태·전환일 기록 테스트
3. breadth 계산이 dataSufficient 제외를 정확히 반영 (짧은 데이터 종목 혼입 픽스처)

### US-3. 진입가 엔진 (1/2) — 피벗 산정 + 파생 가격
- [x] **개발자로서, 설계서 §2~§3의 피벗과 가격 세트를 결정적으로 산출하는 함수를 원한다.**

**구현:** `src/lib/entryPoint.js` + `constants/entry.js`
- 선행 고점 `PH_d`(당일 제외 63일 max close), 돌파 이벤트(상향 교차) 탐지,
  피벗 결정 3규칙(머리 위 저항 / 최근 21일 내 돌파된 저항 / 저항선 소멸) — 설계서 §2.1 그대로
- 파생 가격: 트리거 T = P×1.003, 유효 상단 U = P×1.05, 손절 참고 2종
  (고정 −8% = 기존 STOP_PCT 공유, ATR 비례 = close − 2.5×ATR14 — v7 ATR 재사용), 원거리 경계 P×0.90
- 데이터 63거래일 미만 → 산정 불가 반환

**승인 기준:**
1. 피벗 3규칙 각각의 픽스처 테스트 (교차일 정확성, 21일 경계, 연속 상승 → 소멸) — 6개 이상
2. 당일 제외 규칙 테스트 (당일이 신고가여도 PH에 미포함)
3. 가격 세트가 상수 모듈 값으로만 계산됨 (매직넘버 0)

### US-4. 진입가 엔진 (2/2) — 상태 판정 + 거래량 확인 + 경과일
- [x] **사용자로서, 종목이 4개 진입 상태 중 어디에 있고 돌파의 질이 어떤지 알고 싶다.**

**구현:** `src/lib/entryPoint.js` 확장

- 거래량 확인: 돌파일 volume ≥ 1.5×SMA50(volume) → volumeOK. 미충족 시 상태 2 유지 + 경고 플래그
- 🆕 v9.1 반영: 상태 2에 **돌파 후 경과 거래일** 포함, 0~1일이면 `earlyBreakout: true`

**승인 기준:**
1. 상태 경계 테스트: P×0.90 / P / U 정확 경계, 갭업 U 초과 → 상태 3 — 6개 이상
2. 경과일·earlyBreakout·volumeOK 조합 테스트 4개 이상
3. 엣지 케이스(설계서 §8): 산정 불가·저항 소멸·무거래량 돌파 각 1개 이상

### US-5. 청산 신호 계층 1 (종목 수준)
- [x] **사용자로서, 보유·관심 종목의 추세 건강 이상 신호를 기계적 판정으로 보고 싶다.**

**구현:** `src/lib/exitSignals.js` + `constants/exit.js`
- X1~X5 판정 (PRD §4.4 표 그대로): 50일선 이탈(+거래량 강도), 데드크로스(최근 5일 내 교차),
  템플릿 붕괴(미충족 ≥3, T1/T5 포함 시 강 — 기존 minervini 템플릿 판정 재사용, 재구현 금지),
  클라이맥스 런(10일 수익률 ≥+25% AND SMA50 이격 ≥+25%), 돌파 후 최대 낙폭일(최근 5일 내 + 거래량)
- 반환: 신호 배열 [{code, strength, evidence}] — 권유 표현 없는 사실 기술만

**승인 기준:**
1. 신호별 경계 테스트 (교차 시점, 임계 정확 경계) — 10개 이상
2. X3이 기존 minervini.js 템플릿 함수를 호출함이 확인된다 (재구현 없음)
3. evidence 문자열이 "조건 충족" 프레임이며 매수/매도 권유 표현이 없다 (금칙어 테스트)

### US-6. 포지션 계획 계층 2 (포지션 수준)
- [x] **사용자로서, 내 체결가 기준의 청산 계획(R-배수·손절·트레일링·알림)을 보고 싶다.**

**구현:** `src/lib/positionPlan.js`
- 입력: {ticker, entryPrice, entryDate?} + 시계열 → 산출:
  손절 참고 2종(각 검증 상태 라벨 코드 포함), R-배수(현재 수익 ÷ 8%),
  브레이크이븐 알림(≥ +2R), 트레일링 참고(entryDate 있으면 보유 중 최고 종가 × 0.85, 손절과 max 강조),
  이익 보호 알림(최대 +20% 도달 이력 + 현재 이익 ≤ 최대의 50%)
- entryDate 없으면 트레일링·이익 보호는 "체결일 입력 시 제공" 상태 반환
- 검증 상태 매핑: `constants/verification.js` 신규 — 규칙 코드 → {status: "우위|열위|측정중",
  basis: 근거 문구} 정적 매핑 (초기값: 고정 −8% = 열위·"단일 상승 국면 Out 실측",
  ATR 손절 = 측정중, 60일 보유 = 우위). 갱신은 운영자 changelog 규약

**승인 기준:**
1. R-배수·알림 경계 테스트 (+2R 정확 경계, 이익 보호 50% 반납 경계) — 6개 이상
2. entryDate 유무 분기 테스트
3. verification.js에 −8% 열위 상태가 존재하고 positionPlan 출력에 상태 코드가 실린다

### US-11. 백테스트 — NGX 전체 프로토콜

**승인 기준:**

2. 유니버스 하드코딩 없음 테스트 (동일 코드가 두 픽스처에서 각자 내부 기준으로 계산)

### US-13. 화면2 — 국면 배지 · 진입 상태 · 매도 신호 · 검증 상태
- [ ] **사용자로서, 추천 카드에서 국면·진입 가격·매도 신호를 검증 상태와 함께 보고 싶다.**

**구현:** 화면2 확장 (기존 요소 무변경 — 추가만)
- 상단 국면 배지: 현재 국면 + breadth 값 + 해당 국면 Out 조건부 성과 1줄
  (regimeAxis에서 sample:"out"만, 표본 <50이면 "표본 부족 — 참고 불가") +
  고지 "국면 분류는 기계적 지표이며 시장 예측이 아닙니다"
- 카드: 진입 상태 배지(0~3, 상태별 표시 규칙 = 설계서 §5 표) + 가격 세트
  (트리거/유효상단/손절 참고 2종 — 각 검증 상태 라벨) + 돌파 후 경과일 + earlyBreakout 주의 문구 +
  산출 근거 펼침 + 기준일 명시 + 진입가 면책 고정 문구
- 매도 신호 배지: "매도 신호 없음 / ⚠ n건" + 펼침 (신호·근거·검증 상태)
- backtest.json v3 부재·regimeAxis 부재 시 국면 영역 미표시, 나머지 기능 정상

**승인 기준:**
1. 국면 배지 3상태 + 표본 부족 + 고지 동반 렌더링 테스트
2. 진입 상태 4종 카드 렌더링 (상태 3은 가격 미제시 확인) + earlyBreakout 문구 테스트
3. **검증 열위 표기 테스트**: −8% 손절 참고가에 열위 라벨이 항상 동반 (분리 렌더 불가)
4. 신규 파일·필드 부재 시 v9.1 화면과 동일 (graceful degradation) + 기존 테스트 전부 통과
