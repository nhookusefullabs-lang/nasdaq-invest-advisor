## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 나스닥 종목추천 |
| 한 줄 정의 | 기술적 지표 기반으로 나스닥100 종목 중 유망 종목을 추천하고, 3개월 시뮬레이션과 포트폴리오 예상 수익률을 보여주는 웹 서비스 |
| 목적 | 투자 초보자가 복잡한 분석 없이도 "지금 살만한 종목"에 대한 힌트를 얻을 수 있게 한다 |
| 타겟 유저 | 미국 주식(나스닥)에 관심 있는 개인 투자자, 특히 기술적 분석에 익숙하지 않은 초보자 |
| 개발 난이도 목표 | 첫 바이브코딩 프로젝트 — 화면 4개, 백엔드 없는 정적 웹앱. RSI/MACD 등 지표 계산은 클라이언트에서 처리 |

## 2. 배경 및 문제 정의

- 개인 투자자는 나스닥 100개 이상 종목 중 어떤 것을 볼지부터 막막함을 느낀다.
- 이동평균선, 거래량, RSI, MACD 같은 기술적 지표는 존재하지만 이를 한눈에 종합해서 보여주는 도구가 부족하다.
- 원래 아이디어(13F 기관투자 분석, S&P500 포함, 멀티 에이전트 운영 등)는 완성도는 높지만 데이터 소스 확보와 개발 난이도가 높아 1차 버전에서는 제외한다.

## 3. MVP 범위 (MoSCoW)

### Must have
1. **종목 검색 (상세 필터 포함)**: 나스닥100 종목 리스트에서 이름/티커 검색 + 아래 4개 조건 필터링
   - 이동평균선 이격도 (현재가와 20일 이평선의 괴리율)
   - 거래량 증가 여부 (최근 5일 평균 vs 직전 20일 평균)
   - 주도주 섹터 (최근 3개월 섹터별 평균 수익률 상위 섹터)
   - 과열/과매도 상태 (RSI 14일, 70 이상 과열 / 30 이하 과매도)
2. **종목 추천**: RSI·MACD로 매수 신호를 판정한 뒤 이격도·거래량·주도섹터로 순위를 매기는 2단계 추세추종 로직으로 상위 5~10개 추천
3. **추천 이유 표시**: 지표 기반 한 줄 설명
4. **과거 3개월 시뮬레이션**: 추천 종목을 3개월 전에 샀다면 얻었을 실현 수익률 계산
5. **포트폴리오 구성 (균등 비중)**: 선택 종목 3~5개를 균등 비중으로 묶어 종합 예상 수익률·변동성 제시
6. **선택 상태 유지**: 새로고침해도 검색 조건·선택 종목이 유지됨 (localStorage)

### Should have
7. 리밸런싱 시점 알림 (단순 규칙 기반)

### Could have
8. **위험 성향별 포트폴리오 3유형** (안정형/균형형/공격형)
9. 손절/익절 자동 전략 제안
10. S&P500까지 종목 범위 확대
11. 13F 보고서 기반 기관투자자 매수 분석

### Won't have
- 멀티 에이전트 구조
- 실시간 자동매매 및 실제 증권사 계좌 연동 (데이터 조회만)
- 로그인/회원 시스템 (익명 사용, 브라우저 로컬 저장으로 대체)
- 실시간 시세 갱신 (사전 수집 데이터 기준)

## 4. 핵심 기능 상세

### 4.1 종목 검색 (상세 필터)
- 기본: 나스닥100 대상 티커/종목명 검색, 섹터 필터
- **초기 상태**: 홈 화면 첫 진입 시 상세필터 4종은 **모두 꺼짐 = 나스닥100 전체 표시**. 사용자가 원하는 필터만 켜서 좁힌다. 〔🆕 v4〕
- 상세 조건 (조합 가능), 모두 사전 수집 데이터로 **클라이언트에서 직접 계산**:
  - **이평선 이격도**: `(현재가 − 20일 이동평균) / 20일 이동평균 × 100 (%)` — 예: "+5% 이상"만 보기
  - **거래량 증가**: 최근 5일 평균 거래량이 직전 20일 평균 대비 일정 % 이상 증가한 종목만
  - **주도주 섹터**: 아래 정의로 계산한 **상위 3개 섹터**에 "주도 섹터" 태그 부여, 해당 섹터만 필터 〔🔀 2개→3개〕
    - 섹터 수익률 = **섹터 구성종목의 최근 3개월 수익률 균등평균**. 단 **구성 종목이 1개뿐인 섹터는 대표성 부족으로 주도섹터 판정에서 제외**한다. 〔🆕 v4〕
  - **과열/과매도**: RSI(14) 70 이상(과열) / 30 이하(과매도) 필터
- **데이터 부족 종목 처리**: 거래일이 부족하거나(신규 편입 등) 결측이 있어 지표 계산이 불안정한 종목은 **추천·필터 대상에서 제외**하며, 제외 사유는 데이터 수집 로그에 기록한다. 〔🆕 v4〕

### 4.2 종목 추천 (2단계: 신호 판정 → 순위 스코어링)

**1단계 — 매수 신호 판정 (모두 충족해야 통과)**

| 조건 | 기준 |
|---|---|
| RSI(14) | 50 이상 |
| MACD | MACD선이 0선 위 (12일 EMA > 26일 EMA) |
| 골든크로스 | 최근 5거래일 이내 MACD선이 시그널선(MACD의 9일 EMA)을 상향 돌파 |

**2단계 — 순위 점수 (1단계 통과 종목, 100점 만점)**

| 요소 | 배점 | 계산식 (선형·클램프) |
|---|---|---|
| 이평선 이격도 | 0~60 | `min(disparity, 15) / 15 × 60` (disparity ≤ 0이면 0점, +15%에서 만점) |
| 거래량 증가 | 0~30 | `min(volTrend, 50) / 50 × 30` (음수면 0점, +50%에서 만점) |
| 주도섹터 가산 | 0 또는 10 | 주도 섹터(4.1 기준, 상위 3개) 소속이면 10점 |

- 최종 순위: 1단계 통과 종목을 2단계 점수 순 정렬 → 상위 5~10개 추천
- **조건 완화 폴백**: 1단계 통과가 5개 미만이면 단계적 완화 → ① 골든크로스 기준 5일→10일 확대 → ② 그래도 부족하면 MACD(0선 위)는 유지, RSI≥50만 적용. 완화 시 화면에 **"조건 완화 적용됨"** 표시
- **빈 결과 최종 처리**: 위 완화를 끝까지 적용해도 통과 종목이 5개 미만이면, **억지로 채우지 않고 있는 만큼만 표시**하며 **"매수 신호가 충분치 않습니다" 배너**를 노출한다. 〔🔀 v4 — 기존 "최소 5개 확보" 폐기〕
- 결과: 종목 리스트 + 순위 점수 + 추천 이유 한 줄 (예: "RSI 62, MACD 골든크로스, 주도 섹터 소속")

### 4.3 시뮬레이션 & 예상 수익률
- 최근 3개월 가격 데이터 기준 "3개월 전 매수했다면"의 결과 계산 — 미래 예측이 아님을 화면에 명확히 고지: **"과거 3개월 실현 수익률 (미래 보장 아님)"**
- **매수가 앵커 정의**: "3개월 전 종가" = **최근 63거래일 중 첫 거래일의 종가**로 고정한다(달력 날짜가 아닌 거래일 기준이라 항상 존재하고 모든 종목에 동일 적용). 〔🔍 v4 확정〕
- `수익률% = (현재 종가 / 앵커 종가 − 1) × 100`, 기간 내 최고가/최저가 및 변동폭 표시
- 화면에 **"데이터 기준일: YYYY-MM-DD"** 표시 (§7 `generatedAt`). 〔🆕 v4〕

### 4.4 포트폴리오 구성 (균등 비중)
- 추천 종목 중 사용자가 체크박스로 3~5개 선택
- **균등 비중**: 종합 수익률 = 개별 3개월 실현 수익률의 단순 평균
- **종합 위험 지표**: 개별 종목 변동성(일별 수익률 표준편차)의 단순 평균. 화면 라벨은 **"평균 개별 변동성(상관관계 미반영)"** 으로 표기해, 정확한 포트폴리오 변동성이 아님을 명시한다(종목 간 상관관계는 무시 — 첫 프로젝트 단순화). 〔🔍 v4 라벨 확정〕
- 화면에 "안정형/공격형 등 위험성향별 구성은 다음 버전" 안내 + **"데이터 기준일: YYYY-MM-DD"** 표시 〔🆕 v4〕
- (위험성향 3유형은 Could have)

### 4.5 선택 상태 유지
- 검색 조건, 선택 종목, 마지막으로 본 화면을 localStorage에 저장 → 새로고침 시 복원
- **스키마 버전 키**를 함께 저장한다. 데이터를 재수집해 종목 구성이 바뀐 경우, 복원 시 **현재 데이터에 존재하지 않는 티커는 조용히 제거**하고 나머지만 복원한다(버전 불일치 시에도 앱이 깨지지 않도록). 〔🆕 v4〕
- 계정/로그인 없이 "이 브라우저에서만" 유지 (기기 변경 시 초기화됨을 안내)

## 5. 화면 구성 (4개, SPA 구조)

| 화면 | 설명 | 전달받는 상태 |
|---|---|---|
| 1. 홈/검색 | 종목 검색 + 상세 필터 4종(초기 전부 꺼짐) | localStorage 복원 조건 |
| 2. 추천 결과 | 추천 종목 리스트 + 이유 + 체크박스 선택 (+ 필요 시 완화/빈결과 배너) | 1번의 검색/필터 조건 |
| 3. 시뮬레이션 | 선택 종목의 3개월 시뮬레이션 결과 | 2번에서 선택한 종목 |
| 4. 포트폴리오 | 선택 종목 균등 비중 종합 수익률·평균 개별 변동성 | 2번의 선택 종목, 3번 결과 |

- 별도 페이지 이동 없이 하나의 앱에서 전환(SPA). 화면 간 데이터는 최상위 컴포넌트가 보유 후 props로 전달.
- 모든 결과 화면(2·3·4) 하단에 **면책 문구**와 **"데이터 기준일: YYYY-MM-DD"** 고정 표시. 〔🆕 v4〕

## 7. 데이터 소스 (확정)

- **수집 API: `yfinance`** (Python 무료 라이브러리, API키/계좌 불필요)
- **수집 절차**:
  1. `scripts/collect_data.py` 에서 나스닥100 종목의 일별 가격·거래량을 로컬에서 **1회** 수집 (Python, `yfinance`/`pandas`)
  2. **나스닥100 티커·종목명·섹터 리스트는 스크립트 상단에 정적 배열로 하드코딩**
  3. **수집 기간은 약 6개월치** — MACD(26일 EMA)+시그널(9일 EMA) 워밍업(~35거래일)을 확보하기 위함. **지표는 6개월 전체로 계산하되, 시뮬레이션·화면 표시는 최근 3개월(~63거래일)만 사용**
  4. 결과를 `public/data/nasdaq100.json`으로 저장해 repo에 포함
  5. **데이터 부족·결측으로 지표 계산이 불안정한 종목은 수집 단계에서 걸러 제외하고 사유를 로그로 남긴다** 〔🆕 v4〕
  6. 웹앱은 이 JSON만 읽어 검색·추천·시뮬레이션·포트폴리오 계산 수행
- **장점**: API키가 브라우저에 노출되지 않고, 실행 중 백엔드 불필요. 과거 일별 데이터는 고정값이라 재실행해도 사실상 동일
- **한계**: 실시간 갱신 안 됨. 최신화하려면 수집 스크립트 재실행

**JSON 스키마 (확정):**
```json
{
  "generatedAt": "2026-07-04",
  "tickers": [
    {
      "ticker": "AAPL",
      "name": "Apple Inc.",
      "sector": "Technology",
      "series": [
        { "date": "2026-01-06", "high": 0, "low": 0, "close": 0, "volume": 0 }
      ]
    }
  ]
}
```
- `high`/`low`: 시뮬레이션 최고가/최저가 변동폭
- `close`: 지표(RSI/MACD/이격도) 및 수익률
- `volume`: 거래량 추세
- `generatedAt`: 화면 "데이터 기준일" 표기에 사용 〔🆕 v4〕

## 8. 지표 계산 명세 (결정적 재현 위해 방식 고정)

| 지표 | 방식 |
|---|---|
| SMA(20) | 단순이동평균 |
| EMA(12), EMA(26), EMA(9) | 지수이동평균 |
| RSI(14) | **Wilder 평활** 방식 |
| MACD | MACD선 = EMA12 − EMA26, 시그널 = MACD의 EMA9, 히스토그램 = MACD − 시그널 |
| 이격도 | `(현재가 − SMA20) / SMA20 × 100` |
| 거래량 추세 | `(최근5일 평균 − 직전20일 평균) / 직전20일 평균 × 100` |
| 골든크로스 | 최근 N거래일 내 MACD선이 시그널선을 상향 돌파 (기본 5일, 폴백 시 10일) |
| 변동성 | 일별 수익률의 표준편차 (포트폴리오 "평균 개별 변동성" 계산용) |
| 섹터 수익률 | 섹터 구성종목 최근 3개월 수익률의 균등평균 (1종목 섹터 제외) 〔🆕 v4〕 |
| 3개월 앵커 | 최근 63거래일 중 첫 거래일 종가 〔🔍 v4〕 |
