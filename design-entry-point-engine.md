# 설계 문서: 진입가 산정 로직 (Entry Point Engine) — v10 편입용 스펙 초안

> 목적: 추천 종목에 대해 "얼마에 진입할 것인가"를 결정적(deterministic)으로 산출한다.
> 미너비니 SEPA의 Entry Point 개념을 기존 수집 데이터(일봉 OHLCV)만으로 근사한다.
> 설계 제약: ① 같은 데이터 → 항상 같은 가격 (주관 개입 없음) ② 기존 지표 최대 재사용
> ③ 모든 상수는 constants 모듈에 (백테스트 조정 대상) ④ 산출 근거를 화면에서 확인 가능

## 1. 개념 모델

종목은 피벗(직전 저항선) 대비 위치에 따라 4개 진입 상태 중 정확히 하나에 있다.
진입가 엔진의 출력은 "가격 하나"가 아니라 **상태 + 그 상태에 맞는 가격 세트 + 산출 근거**다.

```
        [상태 0]          [상태 1]        [상태 2]           [상태 3]
        베이스 원거리      돌파 대기        매수 유효 구간       확장(추격 금지)
  ──────────┬──────────────┬──────┬───────────────┬──────────────────→ 가격
        P×0.90           피벗 P   트리거 T=P×1.003   상단 U=P×1.05
```

## 2. 피벗 산정 알고리즘

### 2.1 정의 (v1 — 63일 롤링 선행 고점 방식)

```
PH_d  = max( close[d−63 .. d−1] )          # d일의 "선행 고점" (당일 제외)
돌파 이벤트: close_d > PH_d 이고 close_{d−1} ≤ PH_{d−1}   # 상향 교차일만 (연속 상회는 제외)
```

- **피벗 P의 결정**:
  1. `close_t ≤ PH_t` (현재가가 선행 고점 아래) → **P = PH_t** (머리 위의 저항선)
  2. `close_t > PH_t` 이고 최근 21거래일 내 돌파 이벤트 B 존재 → **P = PH_B** (돌파된 저항선)
  3. `close_t > PH_t` 이고 최근 21거래일 내 돌파 이벤트 없음 (돌파 후 장기 상승 지속)
     → 피벗 무효. 상태 3(확장) 강제 배정, 사유 = "직전 저항선 소멸 — 다음 베이스 대기"
- 데이터 63거래일 미만 종목 → 피벗 산정 불가, 진입가 미제시 ("피벗 산정 불가" 라벨,
  기존 데이터 부족 패턴과 동일 처리)

### 2.2 v2 정교화 옵션 (이번 범위 아님 — 백테스트로 가치 확인 후)
- VCP 마지막 수축 파동의 고점을 피벗으로 (파동 인식 필요 — 난이도 높음)
- 종가 대신 고가(high) 기반 선행 고점 (보수적 피벗 — 변형으로 비교 가능)

## 3. 파생 가격 산출

| 가격 | 정의 | 상수 (constants 모듈) |
|---|---|---|
| 트리거가 T | `P × (1 + BUFFER)` | `BUFFER = 0.003` (0.3%) |
| 유효 상단 U | `P × (1 + CHASE_LIMIT)` | `CHASE_LIMIT = 0.05` (5%) |
| 손절 참고가 | `체결가 × (1 − STOP_PCT)` | `STOP_PCT = 0.08` (변형 D와 동일 상수 공유) |
| 원거리 경계 | `P × (1 − FAR_BAND)` | `FAR_BAND = 0.10` (10%) |

- BUFFER의 목적: 피벗 정확히 그 가격에서의 잦은 되돌림(위스커) 회피. 고정 %로 시작하고,
  변동성 비례 버퍼(예: `max(0.3%, 0.15 × ATR14/close)`)는 변형으로 비교 (ATR은 v7에 이미 구현됨)
- **목표가는 제시하지 않는다** — 추세추종·미너비니 공통 철학: 이익은 목표가가 아니라
  추세 소진/청산 규칙으로 마감. 카드에도 "목표가 없음 — 청산 규칙 참조" 명시

## 4. 거래량 확인 (돌파의 질)

```
돌파일 거래량 조건: volume_B ≥ VOL_MULT × SMA50(volume)_B      # VOL_MULT = 1.5
```

- 상태 2 판정 시 돌파 이벤트 B의 거래량을 검사:
  - 충족 → "거래량 동반 돌파 확인" 배지
  - 미충족 → 상태 2 유지하되 **"⚠ 무거래량 돌파 — 신뢰도 낮음"** 경고 배지 (강등 아님, 정보 표시)
- 상태 1(돌파 대기)의 카드 안내문: "트리거가 상향 돌파 시 **당일 거래량이 50일 평균의
  1.5배 이상인지 확인 후** 진입" — 예약 주문은 거래량 조건을 걸 수 없으므로 사용자 확인 사항임을 명시

## 5. 진입 상태 판정 (의사코드)

```
if 데이터 < 63거래일:            return { state: "산정불가" }
P = 피벗(§2.1)
if P == 무효:                   return { state: 3, reason: "저항선 소멸" }
T = P × 1.003;  U = P × 1.05

if close ≤ P:
    if close < P × 0.90:        return { state: 0, trigger: T, distancePct }   # 원거리 관망
    else:                       return { state: 1, trigger: T }                # 돌파 대기
else:  # close > P — 돌파 이후
    B = 최근 돌파 이벤트 (≤21거래일 보장됨, §2.1-2)
    volumeOK = volume_B ≥ 1.5 × SMA50vol_B
    if close ≤ U:               return { state: 2, upper: U, volumeOK, breakoutDate: B }
    else:                       return { state: 3, reason: "유효 구간(+5%) 초과" }
```

- 갭 처리: 돌파일 시가가 이미 U를 초과한 갭업이면 상태 3 (극단 갭은 추격 금지 — 상태 2 판정에
  자동 포함됨: close > U)
- 상태별 카드 표시:

| 상태 | 배지 | 표시 가격 | 안내문 |
|---|---|---|---|
| 0 원거리 | "베이스 진행 중" | 트리거 T, 현재가와 거리 −n% | "피벗까지 n% — 관망" |
| 1 돌파 대기 | "돌파 대기" | **트리거 T**, 손절 참고(T×0.92) | "T 돌파+거래량 확인 시 진입, 선취매 금지" |
| 2 매수 유효 | "매수 유효 구간" (+거래량 배지) | 피벗 P, **유효 상단 U**, 손절 참고(체결가 −8%) | "구간 내 진입 가능, U 초과 추격 금지" |
| 3 확장 | "확장 — 추격 금지" | (가격 미제시) | "다음 베이스 형성 대기" |

- 모든 상태 공통: 산출 근거 펼침 = 피벗 값·산정 기간(63거래일 선행 고점)·돌파일·돌파일 거래량 배율
- **기준일 명시 필수**: "피벗 기준일 YYYY-MM-DD — 이후 가격 변동 미반영, 실시간 확인 필요"
- 강화 면책 (진입가 영역 고정 문구): "진입 참고 가격은 과거 가격 구조에서 기계적으로 산출된
  값이며 매수 권유가 아닙니다"

## 6. 2단계 스코어링과의 관계

- 진입 상태는 순위 점수에 **반영하지 않는다** (v10 시점). 선별(무엇을)과 진입(얼마에)의
  관심사 분리 — 상태는 카드에 병기되는 직교 정보
- 단, "상태 3(확장) 종목을 top5에서 제외" 정책은 유력한 후보 변형으로 등록한다
  (`entry_exclude_extended`) — 신선도 문제의 직접 해법일 수 있음. 채택은 Out 성과로 판정

## 7. 백테스트 검증 설계 (진입 규칙 변형)

v9.1의 경로 의존 인프라(exits.mjs) 대칭 구조로 `entries.mjs` 추가:

| 변형 | 진입 규칙 | 비교 목적 |
|---|---|---|
| `entry_close` (현행 기준선) | 신호일 종가 | — |
| `entry_pivot_trigger` | 신호일 이후 21거래일 내 고가 ≥ T인 첫날, 체결가 = max(T, 당일 시가) | 트리거 진입의 순효과 |
| `entry_pivot_trigger_vol` | 위 + 당일 거래량 ≥ 1.5×50일 평균 조건 | 거래량 필터의 추가 가치 |

- **체결가 = max(T, 당일 시가)**: 갭업 개장 시 T가 아닌 시가에 체결되는 현실 반영 (낙관 편향 방지)
- **미체결 신호 처리**: 21거래일 내 트리거 미도달 시 "미체결" — 별도 집계.
  핵심 지표에 **체결률(fill rate)** 추가: 트리거 진입은 되돌림 없이 날아간 승자를 놓친다.
  성과는 "체결 조건부 초과수익"과 "기회비용 포함 초과수익(미체결=0 처리)" 두 벌로 산출
- 조합 실험(최종 목표): `entry_pivot_trigger_vol + exit_stop8_trail15` vs `entry_close + 60일 보유`
  — 미너비니 완전체 근사 vs 현행 근사의 대결. Out 구간·체결률 병기로 판정
- 채택 결정은 운영자 (기존 원칙)

## 8. 엣지 케이스 목록

1. 상장 63거래일 미만 → 산정 불가 (§2.1)
2. 돌파 후 장기 상승 지속으로 교차 이벤트가 21거래일 밖 → 상태 3 강제 (§2.1-3)
3. 갭업으로 유효 상단 초과 개장 → 상태 3 (§5)
4. 무거래량 돌파 → 상태 2 + 경고 배지 (§4)
5. 피벗과 52주 신고가가 일치하는 경우 (사상 최고가 돌파) → 정상 처리, "신고가 돌파" 정보 배지 추가 가능 (Could)
6. 저유동성·이중상장 (NGX 대비) → 유니버스 가드레일에서 선처리, 진입가 엔진은 관여하지 않음
7. 데이터가 수정주가(adjusted) 기준임을 전제 — 분할·배당 조정 일관성은 수집 계층 책임

## 9. 상수 요약 (constants 모듈 등재, 전부 "백테스트 조정 예정 설계값")

| 상수 | 값 | 근거 |
|---|---|---|
| `PIVOT_LOOKBACK` | 63거래일 | 기존 pivotProximity와 동일 (약 3개월 베이스) |
| `BREAKOUT_RECENCY` | 21거래일 | 돌파 유효 기간 (약 1개월) |
| `BUFFER` | 0.3% | 위스커 회피 소폭 버퍼 |
| `CHASE_LIMIT` | 5% | 미너비니 buyable range 관례 |
| `VOL_MULT` | 1.5배 | 거래량 동반 돌파 관례적 문턱 |
| `STOP_PCT` | 8% | 미너비니 손절 원칙 (변형 D와 공유) |
| `FAR_BAND` | 10% | 원거리/대기 경계 |

## 10. 미결 사항 (v10 PRD 확정 시 결정)

1. 추세추종 모드에도 진입가를 제시할지, 미너비니·통합 뷰 전용으로 시작할지
   (권고: 전 모드 동일 적용 — 피벗은 모드 무관한 가격 구조 정보)
2. 상태 3 종목의 top5 제외를 v10에서 변형 측정만 할지, 표시 정책(순위 유지 + 강한 경고)으로
   먼저 갈지 (권고: 측정 먼저, 표시는 경고까지만)
3. 포트폴리오/시뮬레이션 화면과의 연동 (진입가·손절가를 시뮬레이션 기본값으로 쓸지) — v10 범위 밖 제안
