## 2. 배경 — v9.1 진단이 가리킨 방향

세 가설의 판정이 하나로 수렴했다: **성과는 "어디서(피벗), 어떤 조건(확인·거래량·국면)에서 진입했는가"가 결정한다.**
- 신선도 역전(돌파 당일 +2.09%p vs 3~4일 확인 후 +10.16%p, Out·60일): 확인된 돌파가 이긴다 → 진입가 엔진에 확인일 개념 필요, 매일 운영 기각
- **돌파 이력 부재 코호트의 열위 (보너스 발견)**: 최근 돌파 이력 없이 추천된 신호는 순위가 높아도 열위 — 미너비니 no_recent_breakout n=289 중앙값 −1.49%(돌파 코호트 대비 확연히 낮음), 추세추종 n=5 −29%p → 진입 상태 분류(상태 0·1·2·3)와 "실행 가능 상태만 top5" 정책 후보의 직접적 실증 근거
- 변형 D 기각(손절 적중률 59%가 승자를 갈라먹음): 진입 정밀도 없이 타이트한 손절은 역효과 → 진입 변형과 청산 변형의 조합 실험이 정당한 시험
- 완화 신호(In −3.95%p vs Out +14.70%p): 국면 조건부 정책 대상 → 국면 체계 없이는 결정 불가
- 유니버스 한계(펀더멘털 Pass n=10, 초대형주의 완만한 돌파): NGX 파일럿으로 상방 탐색
- In/Out 역전의 재해석: "기간"이 아니라 "국면"의 지식으로 전환 필요 → 국면별 분해

## 4. 핵심 기능 상세

### 4.1 NGX 파일럿 (측정 전용)

**티커 확보**: QQQJ(Invesco NASDAQ Next Gen 100 ETF) 보유 종목 공시 CSV → `collect_data.py`에 유니버스 인자 추가 (`--universe=ndx|ngx`). 주 1회 동일 실행에서 두 유니버스 순차 수집.

**유동성 가드레일 (수집 단계, 상수 모듈 등재)**:
- 최소 주가: $10 이상 (미너비니 스크리닝 관례)
- 최소 거래대금: 20일 평균 거래대금 $20M 이상
- 미달 종목은 `excluded[]`에 사유 기록 (기존 패턴)

**예외 처리**: 이중상장(비-나스닥 라인 제외, 티커 정규화), 상장 252거래일 미만(기존 hasFullYearData 제외 — NGX에서 제외율이 높을 것을 수집 로그에 집계·명시)

**측정 프로토콜**: 나스닥100과 동일한 백테스트 전 축(전략·컨센서스·펀더멘털·국면·진입/청산 변형)을 NGX 자체 등가중 벤치마크 대비로 실행 → `backtest_ngx.json`. RS 백분위는 NGX 유니버스 내에서 독립 계산.

**채택 판정 기준 (선커밋)**: NGX의 ★★ 컨센서스가 ① In/Out 양쪽에서 나스닥100 ★★ 대비 초과수익 우위, ② 펀더멘털 Pass 표본 ≥ 50건(현행 10건의 5배), ③ Out 표본 ≥ 100건일 때 → v11에서 유니버스 사용자 옵션으로 노출 검토. 미달 시 파일럿 결과만 기록하고 보류.

### 4.2 시장 국면 체계 (사다리 1~3단)

**국면 지표 (결정적, 기존 데이터만 사용)**:
```
breadth_t = (유니버스 중 close > SMA200 종목 비율)      # dataSufficient 종목 기준
상태 전이 (히스테리시스):
  상승 → 중립: breadth < 0.55        중립 → 상승: breadth > 0.65
  중립 → 하락: breadth < 0.40        하락 → 중립: breadth > 0.50
초기 상태: breadth ≥ 0.55 → 상승 / 0.40~0.55 → 중립 / < 0.40 → 하락
```
- 상수 4종은 constants 모듈 (`REGIME_*`), "설계값 — 국면별 분해로 사후 검증" 주석
- 국면은 주간 데이터 갱신 시 자동 재계산 (별도 수집 불요)

**1단 — 국면별 성과 분해 (백테스트)**: 모든 신호를 신호일의 국면으로 귀속 → 전략×국면×보유 성과표를 backtest.json에 추가 (`regimeAxis`). In/Out 역전을 국면 라벨로 재해석하는 것이 1차 목표.

**2단 — 화면 표시**: 화면2 상단 국면 배지 ("현재 시장 폭 n% — 상승/중립/하락 국면") + 해당 국면의 Out 조건부 성과 1줄 ("이 국면에서 ★★ 컨센서스의 검증 초과수익: +n.n%p · 표본 n건"). 표본 부족 국면(예: 하락 표본 희소)은 "표본 부족 — 참고 불가" 정직 표기. 고정 고지 문구 유지 + "국면 분류는 기계적 지표이며 시장 예측이 아닙니다" 추가.

**3단 — 소프트 정책 변형 (측정만, Should)**: §3 Should 3종. 채택 판정 기준(선커밋): 해당 국면 Out 표본 ≥ 50건에서 초과수익·승률 모두 현행 이상. **완화 폴백의 국면 조건부 비활성이 최우선 검증 대상** (v9.1 가설 ① 후속).

**4단(하드 정책)은 명시적 제외** — 국면 전환 표본 부족(3년 내 1~2회)으로 검증 불가.

- 미결 사항 확정: 진입가는 **전 모드 동일 제시** (설계서 §10-1 권고안 채택), 상태 3 top5 제외는

| X5 | 돌파 후 최대 낙폭일 | 돌파 이후 최대 낙폭 거래일이 최근 5거래일 내 + 거래량 동반 | 경고 |

- 카드 배지: "매도 신호 없음 / ⚠ 매도 신호 n건" + 펼침에 신호별 판정 근거와 **검증 상태**
- 프레임 원칙: "규칙 충족의 기계적 표시"만 ("50일선 이탈 조건 충족" ○ / "지금 파세요" ×) — 리서치 배지의 권유 금지 원칙과 동일선

**계층 2 — 포지션 수준 청산 계획 (화면 3·4)**

- 입력: 체결가 (필수), 체결일 (선택) — localStorage v4에 저장
- 산출 (상수는 진입가 엔진과 공유):
  - 손절 참고가 2종 병기: 체결가 × 0.92 / 체결가 − 2.5×ATR14 (각 검증 상태 표기)
  - **R-배수**: 현재 수익 ÷ 초기 리스크(8%) — "현재 +1.7R"
  - **브레이크이븐 상향 알림**: +2R 도달 시 "손절선을 체결가로 상향 검토" 표시
  - **트레일링 참고가**: 보유 중 최고 종가 × 0.85 (체결일 입력 시) — 손절선과 트레일링 중 높은 쪽 강조
  - **이익 보호 알림**: 최대 +20% 이상 도달 이력 + 현재 이익이 최대치의 절반 이하로 반납 시
- 사용 안내 고정 문구: "빠른 방어는 증권사 예약 주문(stop-loss)으로 — 본 시스템은 주 1회 갱신되므로 실시간 감시를 대체하지 않습니다"
- 강화 면책: "청산 참고 정보는 기계적 규칙의 산출값이며 매도 권유가 아닙니다"

### 4.5 백테스트 확장

**진입 변형 (`entries.mjs`)**: `entry_close`(기준) / `entry_pivot_trigger` / `entry_pivot_trigger_vol` / 🆕 `entry_pivot_confirm2`(돌파 후 2거래일 종가가 피벗 위 유지 확인 시 3일째 진입 — 신선도 역전의 직접 검증). 체결가 = max(트리거가, 당일 시가), 체결률·기회비용 포함 성과 병산 (설계서 §7)

**청산 변형 (신규 3종)**: `exit_stop_atr`(2.5×ATR14 손절 + 60일) / `exit_sma50_break`(X1 신호 청산) / `exit_climax`(X4 발생 시 청산). 기존 D 2종 결과는 비교 기준으로 유지

**조합 실험**: `entry_pivot_confirm2 × exit_stop_atr` 및 `entry_pivot_trigger_vol × exit_sma50_break` 등 대표 조합 3개 이내 (격자 자제) — "미너비니 완전체 근사 vs 현행" 최종 대결

**국면별 분해**: 전 축 × 국면 (regimeAxis). **NGX**: 전체 프로토콜 별도 실행 → `backtest_ngx.json`

**backtest.json 스키마 v3**: regimeAxis, entryVariants(체결률 포함), 조합 레코드 추가. 로더 v1·v2 하위 호환.
