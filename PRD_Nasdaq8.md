## 2. 배경 및 문제 정의

- v7까지의 추천은 순수 추세추종(RSI·MACD·이격도) 단일 관점이다. 이격도가 클수록 고득점하는 구조는 "고점 추격 매수"를 유도할 수 있고, 배점(60/30/10)은 검증되지 않은 임의 상수다.
- 마크 미너비니의 SEPA 방법론은 규칙이 정량화되어 있어(트렌드 템플릿 8조건, VCP) 현재의 정적 JSON + 클라이언트 계산 아키텍처에 정합성이 높다. 특히 "추세는 확립됐지만 변동성이 수축된 돌파 직전 종목"을 찾는 관점은 기존 추세추종과 상호 보완적이다.
- 두 모드는 철학이 충돌하는 지점이 있으므로(추세추종=이미 오르는 중 / 미너비니=조용히 준비 중) 원점수 단순 합산은 "어디서도 1등이 아닌 종목"을 통합 1위로 만드는 평균의 함정에 빠진다. → 컨센서스 등급 + 백분위 순위 평균으로 설계한다.
- 두 모드 모두 기술적 분석이므로 펀더멘털(EPS·매출·마진·ROE)과 재료(뉴스·이벤트·공시)의 보완이 필요하다. 펀더멘털은 전 종목에 공평한 **정량 허들**로, 리서치 서브에이전트는 커버리지가 제한적(세션당 최대 15종목)이므로 **소프트 게이트(배지)**로 역할을 분리한다.

## 4. 핵심 기능 상세

### 4.1 추세추종 모드 (기존 v7 — 변경 없음)
- 1단계: RSI≥50 + EMA12>EMA26 + 골든크로스(기본 5일) / 완화 폴백·프리셋(기본형/보수형/공격형/custom)·고급 설정 모두 v7 그대로
- 2단계: 이격도 60 + 거래량 30 + 주도섹터 10 (배점 고정)
- **v7 프리셋·고급 설정은 추세추종 모드에만 적용**된다. 미너비니 모드는 원전 기준 고정(파라미터 조정 없음)

### 4.2 미너비니 모드 (신규)

**1단계 — 트렌드 템플릿 (8조건, Stage 2 판정)**

| # | 조건 | 계산 정의 |
|---|---|---|
| T1 | 현재가 > SMA150 **그리고** > SMA200 | 최신 종가 기준 |
| T2 | SMA150 > SMA200 | 〃 |
| T3 | SMA200이 최소 1개월 상승 추세 | `SMA200(당일) > SMA200(22거래일 전)` |
| T4 | SMA50 > SMA150 > SMA200 | 〃 |
| T5 | 현재가 > SMA50 | 〃 |
| T6 | 현재가 ≥ 52주 최저가 × 1.30 | 52주 = 최근 252거래일 (v7 `week52HighLow` 재사용) |
| T7 | 현재가 ≥ 52주 최고가 × 0.75 | 〃 |
| T8 | RS 백분위 ≥ 70 (유니버스 내 상위 30%) | §8 RS 정의 |

- **완화 폴백**: 8조건 전부 충족 종목이 5개 미만이면 **7/8 충족까지 허용**하되, 카드에 미충족 조건을 명시 표시 (예: "T7 미충족 — 52주 고점 대비 −28%"). 그래도 부족하면 있는 만큼만 + 배너
- **빈 결과 배너**: 기존 `insufficientSignal` 배너 재사용 + 안내 문구 추가: **"미너비니 방법론에서는 조건 미충족 시 현금 보유가 원칙입니다"**

**2단계 — VCP 근사 스코어링 (1단계 통과 종목, 100점 만점)**

| 요소 | 배점 | 계산식 (선형·클램프) |
|---|---|---|
| RS 백분위 | 0~40 | `(RS백분위 − 70) / 30 × 40` (70에서 0점, 100에서 만점) |
| 변동성 수축 | 0~25 | 수축비 = 최근 10일 일수익률 표준편차 ÷ 이전 40일 표준편차. `max(0, (1 − min(수축비, 1)) / 0.5) × 25` (수축비 0.5 이하에서 만점, 1 이상 0점) |
| 거래량 드라이업 | 0~15 | 드라이업% = (최근 5일 평균 거래량 − 50일 평균) / 50일 평균 × 100. 음수(감소)일수록 가점: `min(|드라이업%|, 30) / 30 × 15` (양수면 0점) |
| 피벗/신고가 근접 | 0~20 | 근접% = (최근 63거래일 최고 종가 − 현재가) / 최고 종가 × 100. `max(0, (10 − min(근접%, 10)) / 10) × 20` (고점 일치 시 만점, −10% 이상 이격 시 0점) |

- 배점은 v8에서는 설계값(임시)이며, **v9 백테스트로 실측 조정 예정**임을 코드 주석과 PRD에 명시
- 추천 이유 한 줄: "Stage 2 추세, RS 상위 n%, 변동성 수축 중, 피벗 −n%" 형식

### 4.3 컨센서스 랭킹 (통합 뷰 — 추천 화면 기본)

- **원점수 합산 금지** (두 모드의 점수 분포·척도가 달라 왜곡 발생)
- 각 모드에서 종목의 **백분위 순위**(1단계 통과 종목 중 상위 몇 %)를 계산 → 두 백분위의 단순 평균으로 정렬
- **컨센서스 등급**:
  - ★★ 컨센서스: 두 모드 1단계 모두 통과 — 최상위 그룹으로 우선 표시
  - ★ 단일 신호: 한쪽만 통과 — 어느 모드의 신호인지 라벨 표시
  - 미통과: 추천 제외
- 한쪽 모드만 통과한 종목의 통합 정렬: 통과한 모드의 백분위만 사용하되 ★★ 그룹 아래에 배치
- 카드 표시 예: `NVDA ★★ 컨센서스 — 추세추종 92점 · 미너비니 85점 · 통합 상위 3%`
- 가중 평균(모드별 가중치)은 v9 백테스트에서 컨센서스/단일 신호 승률 실측 후 도입 검토

### 4.4 펀더멘털 허들 (정량, 전 종목 공평 적용)

**판정 기준 (미너비니 원전 기준 — v9 백테스트로 조정 예정)**

| # | 기준 | 임계값 | 데이터 소스 (yfinance) | 판정 반영 |
|---|---|---|---|---|
| F1 | 분기 EPS 성장률 (전년 동기 대비) | ≥ +20% | quarterly income statement | **핵심** |
| F2 | EPS 성장 가속 (최근 분기가 직전 분기보다 성장률 높음) | boolean | 〃 (분기 2~3개 비교) | 참고 배지 |
| F3 | 분기 매출 성장률 (전년 동기 대비) | ≥ +20% | quarterly revenue | **핵심** |
| F4 | 영업마진 개선 추세 (최근 분기 ≥ 직전 분기) | boolean | 영업이익/매출 분기 비교 | 참고 배지 |
| F5 | ROE | ≥ 17% | `info.returnOnEquity` | **핵심** |

**3단계 판정 (탈락이 아닌 강등/표시 — 나스닥100 대형주 특성상 이분법은 빈 결과 위험)**

| 판정 | 조건 | 화면 처리 |
|---|---|---|
| **Pass** | 핵심 3종(F1·F3·F5) 모두 충족 | 등급 배지 + ★★와 결합 시 최상위 표시 |
| **Partial** | 핵심 중 1~2개 충족 | 순위 강등 없음 없이 **컨센서스 순위 그대로 두되** 미충족 항목 명시 (예: "EPS +31% ✓ · 매출 +12% ✗ · ROE 22% ✓") |
| **Fail** | 핵심 0개 충족 | 추천 리스트 하단 "펀더멘털 미달" 접이식 섹션으로 분리 (숨기지 않고 사유와 함께 표시) |

**결측 처리 (기존 dataSufficient 패턴 재사용)**
- 개별 기준의 데이터 결측 시 해당 기준은 "판정불가"로 처리 — 충족으로도 미충족으로도 세지 않음
- 핵심 3종 중 2개 이상 판정불가면 판정 자체를 유보하고 **"펀더멘털 데이터 부족"** 라벨 표시 (Fail로 처리하지 않음)
- 결측 사유는 수집 로그(exclusion_log 패턴)에 기록

### 4.5 리서치 최종점검 배지 (정성, 소프트 게이트)

- **research.json 스키마 v2**: 기존 v1 필드 유지 + item에 `riskFlags: [{type, description}]` 배열 추가 (type 예: `earnings_imminent`, `litigation`, `regulatory`, `guidance_cut`, `other`). `schemaVersion: 2`. v1 파일 로드 시 riskFlags는 빈 배열로 간주 (하위 호환)
- **3상태 배지** (추천 카드):
  - **리서치 점검 ✓**: 리서치 완료 + riskFlags 0건 + sentiment ≠ negative
  - **⚠ 리스크 플래그 n건**: riskFlags 존재 또는 sentiment = negative — 자동 탈락 없이 경고 + 펼치면 사유 표시
  - **리서치 미실시**: research.json에 해당 티커 없음 — 불이익 없음 (커버리지 비대칭 방지)
- **사용자 토글**: "리스크 플래그 종목 숨기기" (기본 꺼짐) — 판단 권한은 시스템이 아닌 사용자에게
- 기존 원칙 유지: "AI가 수집한 참고 정보이며 투자 판단의 근거가 아닙니다" 문구, 매수/매도 권유 표현 금지, stale 경고(`basedOnDataOf` 불일치)
- 리서치 서브에이전트 운영 매뉴얼(`.claude/commands/research.md`)에 riskFlags 작성 규칙 1개 절 추가

### 4.6 추천 파이프라인 (전체 그림)

```
[1] 기술적 신호   : 추세추종 / 미너비니 각 모드 1단계 판정 (완화 폴백 각자 적용)
[2] 컨센서스 랭킹 : 백분위 순위 평균 정렬 + ★★/★ 등급

[4] 리서치 점검   : ✓ / ⚠ 플래그 / 미실시 배지 — 소프트 게이트 (자동 탈락 없음)
```
