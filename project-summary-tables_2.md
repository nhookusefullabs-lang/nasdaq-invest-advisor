## 4. 확정된 설계·운영 원칙

| 원칙 | 내용 |
|---|---|
| 측정 → 판정 → 채택 | 모든 변경은 변형으로 측정, 판정 기준은 결과 확인 전 선커밋, 채택은 운영자 승인 + changelog |
| 같은 코드 원칙 | 백테스트는 앱 lib를 직접 import — 판정 로직 재구현 금지 |
| 통계 정직성 | Out-of-Sample만 화면 표시 · 겹침 보정 유효 표본 병기 · 표본 부족 시 "판단 보류" |
| 검증 상태 표기 | UI 노출 규칙은 우위/열위/측정 중 3상태 병기 (−8% 손절 = 열위 정직 표기) |
| 자동 최적화 금지 | 격자 탐색·자동 적용 코드 자체를 금지 (과최적화 방지) |
| 운영 주기 | 주 1회 토요일 확정 (신선도 역전으로 정당화) · 시간 민감 규칙은 가격으로 제공 + 증권사 예약 주문 위임 |
| 기존 동작 보존 | 릴리스는 추가만 · graceful degradation (신규 파일 부재 시 해당 기능만 비표시) |
| **선별·실행 관심사 분리** | 모드(추세추종/미너비니)는 "무엇을", 실행 계층(돌파 트랙·눌림목 트랙)은 "언제·얼마에" — **거래량 동반 확인(1.5×50일 평균)은 양 트랙 공용 실행 문법**. 게이트(미확인 종목 배제)·스코어 반영은 데이터가 부결/금지 — 실행 지침·표시로만 이식 |
| 한계 고지 | 생존 편향 · 비용 미반영 · 국면 편중 · 펀더멘털 선견 편향 등을 화면·문서에 고정 문구로 |

## 5. v11 확정 구성 (측정 릴리스)

| 항목 | 내용 | 비고 |
|---|---|---|
| 데이터 확장 | 수집 기점 2021-01-01 (약 5.5년) — 2022 약세장 + 온전한 사이클 포함 | 10년은 생존 편향 대비 한계 효용 체감으로 보류. 크기 ~8-11MB/유니버스 |
| **눌림목 진입 변형** | `pullback_resume_vol`: 눌림(피벗 −10~−25%) + 지지(>SMA200/50) + 거래량 고갈 → 재개 트리거(10일 고가 돌파) + 거래량 1.5× — **3종 세트 비교** (즉시 / 재개만 / 재개+거래량) | 발견 #6+#8의 매핑. 이론: 모멘텀∩단기반전, 포켓 피벗, 와이코프 재매집 |
| 청산 후보 A | 국면 조건부 청산 (상승=60일 보유, 하락 전환 시 방어 청산 + regime-flip 즉시 청산 버전) | 발견 #10+#2 |
| 청산 후보 C | 구조 기반 손절 (피벗×0.97 / 눌림 저점×0.98) — 눌림목 변형과 세트 | 미너비니 원전 방식 |
| 청산 후보 E | 클라이맥스 부분 청산 (절반 청산·절반 보유) | 유일한 인프라 확장: 부분 포지션 가중 평균 |
| 청산 후보 F | 보유 지평 90/120일 기준선 추가 | 저비용 — "청산을 늦추는 게 최고의 청산인가" |
| NGX 허들 결합 | ★★ ∩ 펀더멘털 Partial 이상 교집합의 중앙값 검증 → 통과 시 조건부 노출 재판정 | 발견 #9의 힌트 |
| 상태 0 × 국면 분해 | 상태 축을 국면 2차원으로 — 구조의 몫 vs 국면(V자 회복)의 몫 분리 | 발견 #6의 정밀화 |
| relax_off 채택 반영 | 앱 적용 + changelog("표본 박약 — 분기 재검증 조건부") | 운영자 확정 필요 |
| **① 즉시 반영: 거래량 확인 표시 승격** | 검증 상태를 "측정 중" → "조건부 품질 우위(승률 +11.6pt 실측 · 전량 자동 적용 시 기회비용 존재)"로 승격 + 양 모드 카드에서 "거래량 동반 돌파 확인 ✓"를 1급 정보로 격상 | 로직 무변경 — 운영자 결정만으로 가능 (v11 UI 작업에 포함) |
| **② 측정 추가: 진입 변형 모드별 분해** | entryVariants에 전략 축 추가 — 미너비니 모드·★★ 컨센서스 풀에서 거래량 돌파 진입의 효과 측정 (현재는 추세추종 top5 기준만 실측된 공백) | 집계 확장 수준 — 저비용 |
| **③ 측정 후 검토: 트랙 기반 표시 정책** | 돌파 트랙(피벗 트리거+거래량)·눌림목 트랙(재개 트리거+거래량)이 모두 검증되면, 추천 화면을 "순위 + 트랙별 실행 계획" 구조로 재편 검토 — 순위 불변, 실행 정보의 구조화 | 눌림목 변형 판정 이후의 후속 결정 |
