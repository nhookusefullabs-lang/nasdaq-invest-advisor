# PRD: 나스닥 종목추천 (MVP)

## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 나스닥 종목추천 |
| 한 줄 정의 | 기술적 지표 기반으로 나스닥100 종목 중 유망 종목을 추천하고, 간단한 포트폴리오와 예상 수익률을 보여주는 웹 서비스 |
| 목적 | 투자 초보자가 복잡한 분석 없이도 "지금 살만한 종목"에 대한 힌트를 얻을 수 있게 한다 |
| 타겟 유저 | 미국 주식(나스닥)에 관심 있는 개인 투자자, 특히 기술적 분석에 익숙하지 않은 초보자 |
| 개발 난이도 목표 | 첫 바이브코딩 프로젝트 — 화면 4개, 핵심 로직 1~2개로 완결되는 수준 |

## 2. 배경 및 문제 정의

- 개인 투자자는 나스닥 100개 이상 종목 중 어떤 것을 볼지부터 막막함을 느낀다.
- 이동평균선, 거래량 같은 기술적 지표는 존재하지만 이를 한눈에 종합해서 보여주는 도구가 부족하다.
- 원래 아이디어(13F 기관투자 분석, S&P500 포함, 멀티 에이전트 운영 등)는 완성도는 높지만 데이터 소스 확보와 개발 난이도가 높아 **1차 버전에서는 제외**한다.

## 3. MVP 범위 (MoSCoW)

### Must have (반드시 포함)
1. **종목 검색**: 나스닥100 종목 리스트에서 이름/티커로 검색
2. **종목 추천**: 이동평균선 + 거래량 추세 기반 스코어링으로 상위 5~10개 종목 추천
3. **추천 이유 표시**: 왜 추천됐는지 지표 기반으로 간단히 설명 (예: "20일 이평선 상향 돌파, 거래량 증가 중")
4. **과거 3개월 시뮬레이션**: 추천 종목을 3개월 전에 샀다면 얻었을 예상 수익률 계산 및 표시
5. **간단 포트폴리오 구성**: 추천 종목 중 사용자가 선택한 3~5개를 균등 비중으로 묶어 예상 수익률 제시

### Should have (여유 있으면 포함)
6. 포트폴리오 유형 2~3개 비교 제시 (예: 안정형 / 공격형)
7. 리밸런싱 시점 알림 (단순 규칙 기반, 예: "20% 이상 등락 시 검토 권장")

### Could have (다음 버전)
8. 손절/익절 자동 전략 제안
9. S&P500까지 종목 범위 확대
10. 13F 보고서 기반 기관투자자 매수 분석

### Won't have (이번 범위 제외)
- 멀티 에이전트 구조 (투자관리 에이전트 + 서브에이전트 체계) — 로직이 단순한 함수/모듈로 충분한 단계에서는 오버엔지니어링
- 실시간 자동매매 및 실제 증권사 연동
- 로그인/회원 시스템 (익명 사용 가능한 단일 세션으로 시작)

> 원본 기획서의 "가~바" 프로세스와 "6.고려사항"의 멀티 에이전트 아이디어는 좋은 방향이지만, 첫 프로젝트에서는 **하나의 추천 로직 + 하나의 시뮬레이션 로직**으로 압축해 완성 가능성을 높였습니다. 완성 후 여유가 되면 Should/Could 항목부터 순서대로 확장하는 걸 추천합니다.

## 4. 핵심 기능 상세

### 4.1 종목 검색
- 나스닥100 종목 리스트를 대상으로 티커/종목명 검색
- 필터: 섹터 정도만 (기술, 헬스케어, 소비재 등) — 세부 조건은 MVP 제외

### 4.2 종목 추천
- 스코어링 로직(단순화된 기술적 분석):
  - 20일 이동평균선 대비 현재가 위치 (상향/하향)
  - 최근 거래량 증감 추세
  - 위 두 지표를 합산한 간단 점수로 순위 매김
- 결과: 상위 5~10개 종목 리스트 + 점수 + 한 줄 추천 이유

### 4.3 시뮬레이션 & 예상 수익률
- 최근 3개월 가격 데이터 기준, "3개월 전 매수했다면"의 결과를 계산해 보여주는 방식 (실제 미래 예측이 아님을 화면에 명확히 고지)
- 수익률(%), 최고가/최저가 대비 변동폭 표시

### 4.4 포트폴리오 구성
- 추천 종목 중 사용자가 체크박스로 3~5개 선택
- 균등 비중으로 조합한 예상 수익률 계산 (개별 종목 수익률의 단순 평균)

## 5. 화면 구성 (4개 화면으로 압축)

| 화면 | 원본 기획서 대응 | 설명 |
|---|---|---|
| ① 홈/검색 화면 | 가.검색창 | 종목 검색 + 섹터 필터 |
| ② 추천 결과 화면 | 나,다.추천화면 | 추천 종목 리스트 + 추천 이유 + 체크박스 선택 |
| ③ 시뮬레이션/수익률 화면 | 마,바.시뮬레이션·수익율 | 선택 종목의 3개월 시뮬레이션 결과 |
| ④ 포트폴리오 화면 | 라.포트폴리오 화면 | 선택 종목 조합의 종합 예상 수익률 |

> 원본의 "종목 구성을 위한 질문 창"(나. 화면구성)은 이번 버전에서는 별도 화면 대신 ②번 화면 내 체크박스 선택으로 대체했습니다.

## 6. 비기능 요구사항

- **데이터 소스**: 무료 주가 API(예: Yahoo Finance 등) 또는 제공된 더미/샘플 데이터셋 사용 — 실제 API 연동 가능 여부에 따라 조정
- **대상 종목 범위**: 나스닥100 (S&P500은 다음 버전에서 확대)
- **성능**: 종목 데이터는 캐싱하여 반복 계산 최소화 (토큰/API 비용 절감)
- **반응형**: 데스크톱 기준 우선 개발, 모바일은 필수 아님
- **면책 문구**: 모든 추천/시뮬레이션 결과 화면에 "투자 참고용이며 투자 손실 책임은 본인에게 있음" 고지 필수

## 7. 성공 기준 (이번 과제 기준)

- [ ] 종목 검색부터 추천, 시뮬레이션, 포트폴리오 구성까지 4개 화면이 끊김 없이 연결된다
- [ ] 추천 로직이 최소 1개 이상의 실제(또는 샘플) 데이터 기준으로 동작한다
- [ ] 첫 개발 과제로서 로컬에서 데모 가능한 수준으로 완성된다

## 8. 향후 확장 아이디어 (이번 범위 아님, 참고용)

- 투자관리 에이전트 + 서브에이전트(종목검색추천/포트폴리오구성/포트폴리오운영) 체계로 고도화
- 13F 보고서 기반 기관투자자 매수 동향 분석
- 손절/익절 자동 전략 및 리밸런싱 알림 고도화
- S&P500까지 종목 유니버스 확대
