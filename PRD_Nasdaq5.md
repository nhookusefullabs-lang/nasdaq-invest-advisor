## 4. 핵심 기능 상세

### 4.1 종목 검색 (상세 필터)
(v4와 동일 — 변경 없음. §4.1 참고: `PRD_Nasdaq4.md`)

### 4.2 종목 추천 (2단계: 신호 판정 → 순위 스코어링 + 고득점 특별 편입) 🔀🆕

**1단계 — 매수 신호 판정**, **2단계 — 순위 점수(100점 만점)**: v4와 동일한 기준·배점 유지 (`lib/recommend.js`).

**🆕 v5 추가 — 고득점 특별 편입 (High-score inclusion):**
- 1단계 매수 신호를 통과하지 못한 종목이라도, 2단계 점수가 **70점 이상**이면 추천 리스트에 함께 포함한다.
- 이 종목들은 `signalPassed: false`로 구분되며, 화면에는 "고득점 편입" 배지와 "매수 신호 미충족 (고득점 특별 편입)" 사유 문구를 표시해 신호 통과 종목과 명확히 구분한다.
- 신호 통과 종목 + 고득점 편입 종목을 합쳐 점수순으로 정렬한 뒤, 기존과 동일하게 **상위 10개**까지만 노출한다 (`MAX_RESULTS`).
- **"매수 신호가 충분치 않습니다" 경고 배너**(`insufficientSignal`)는 고득점 편입 종목 수와 무관하게, **실제로 매수 신호를 통과한 종목 수만으로** 판단한다 — 배너의 의미(신호 자체가 부족하다는 경고)를 그대로 유지하기 위함.
- 임계값(70점)은 `lib/recommend.js`의 `HIGH_SCORE_INCLUSION_THRESHOLD` 상수로 관리.
- 실제 데이터 확인 사례(2026-07-09): AXON(80.1점), MRNA(77.1점)가 매수 신호는 통과하지 못했지만 고득점 편입으로 리스트 최상단에 노출됨.

### 4.3 시뮬레이션 & 예상 수익률 + 1/3/6개월 미니 차트 🆕

- 3개월 실현 수익률 계산 방식은 v4와 동일 (최근 63거래일 중 첫 거래일 종가를 매수가 앵커로 고정).
- **🆕 v5 추가 — 종목별 1/3/6개월 가격 미니 차트**: 각 선택 종목 카드에 `PriceSparkline` 컴포넌트 3개(1개월/3개월/6개월)를 나란히 표시한다.
  - 데이터 출처: `deriveTickerData()`가 각 종목에 대해 `chart.oneMonth`(최근 21거래일)/`chart.threeMonth`(최근 63거래일, 시뮬레이션 창과 동일)/`chart.sixMonth`(수집된 전체 기간, 약 6개월) 종가 시계열을 함께 계산해 반환한다.
  - 각 미니 차트는 기간 라벨 + 등락률(%, 국내 관례대로 상승=빨강/하락=파랑) + 라인 스파크라인(면적 채움 10% 옵션 포함)으로 구성된 소형 "stat tile" 패턴이다. 개별 데이터 포인트 툴팁은 없음(정보 밀도가 낮은 요약용 차트로 설계).

### 4.4 포트폴리오 구성 (수동 종목 + 수동 비중 + 도넛 차트) 🔀🆕

- **🔀 종목 수 제한 폐지**: 기존 3~5개 제한(`MIN/MAX_PORTFOLIO_SIZE`)을 완전히 없앴다. 1개 이상이면 몇 개든 포트폴리오를 구성할 수 있다.
- **🔀 종목 구성 방식**: 추천 화면 체크박스뿐 아니라, 시뮬레이션·포트폴리오 화면에서도 `TickerPicker`(티커/종목명 검색 + 추가)로 데이터가 충분한(dataSufficient) 종목이면 무엇이든 직접 추가할 수 있고, 각 종목 옆 ✕ 버튼으로 제거할 수 있다. 선택 상태(`selectedTickers`)는 추천/시뮬레이션/포트폴리오 화면이 공유한다.
- **🔀 비중 = 수동 % 입력, 자동 균등 재조정**:
  - 종목을 추가하거나 제거하면 전체 비중이 `100 / 구성종목수`로 균등 재분배된다(디폴트값).
  - 사용자가 한 종목의 비중(%)을 직접 입력하면, **나머지 종목들이 남은 비중(100 − 입력값)을 균등하게 나눠 가져** 합계가 항상 정확히 100%가 되도록 자동 재계산된다.
  - "균등 배분으로 초기화" 버튼으로 언제든 선택을 바꾸지 않고 균등 비중으로 되돌릴 수 있다.
  - 종합 예상 수익률·변동성은 이 가중치를 반영한 **가중 평균**으로 계산한다(`lib/portfolio.js`의 `buildPortfolio(tickers, weights)`). 라벨은 "종합 예상 수익률 (가중 평균)" / "가중 평균 변동성 (상관관계 미반영)".
- **🆕 포트폴리오 도넛 차트** (`PortfolioPieChart.jsx`):
  - SVG로 직접 그린 도넛 차트(차트 라이브러리 미사용). 종목마다 고정 카테고리 색상(8색, `lib/portfolioColors.js`)을 배정하고, **9번째 종목부터는 색상 정체성을 접어 회색 "기타" 조각 하나로 묶는다** (카테고리 색상 8개 초과 금지 원칙).
  - 각 조각에는 **티커를 리더 라인과 함께 항상 직접 라벨링**하고, 마우스오버/키보드 포커스 시 뜨는 툴팁에 **정확한 비중(%)과 실제 회사명**을 함께 보여준다.
  - 비중 입력을 조정하면 차트가 즉시 다시 그려진다(실시간 반영).
  - 비중 입력 목록의 각 행에도 차트와 동일한 색상 점을 표시해 파이차트와 목록이 시각적으로 연결되도록 했다.

### 4.5 선택 상태 유지
- v4와 동일하게 localStorage에 저장하되, **비중(`weights`) 상태도 함께 저장**하도록 확장했다. 스키마 버전은 그대로(1)이며, 저장된 데이터에 `weights`가 없거나 현재 데이터에 없는 티커의 비중 항목은 로드 시 조용히 무시/제거된다.
