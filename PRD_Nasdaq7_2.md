# PRD: 나스닥 종목추천 v7 — 지표 필터 확장 + 추천 프리셋 (계획 문서)

> v5(as-built) 메인시스템과 v6(리서치 서브에이전트, 진행 중)에 이어,
> **화면1 기술적 지표 필터 확장**과 **추천 로직 프리셋**을 추가하기 위한 설계 문서다.
> 원칙: 기존 동작은 변경하지 않고 추가한다. 단 §4.1의 데이터 수집 기간 변경은 예외적 선행 작업이다.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 범위 한 줄 정의 | ① 화면1에 신규 기술적 지표 필터 5종(볼린저밴드, 52주 신고가/신저가, 스토캐스틱, ATR, OBV) 추가, ② 추천 로직을 3종 프리셋(보수형/기본형/공격형)으로 선택 가능하게, ③ 프리셋 파라미터를 직접 조정하는 접이식 "고급 설정" 제공 |
| 목적 | 검색 단계에서 더 다양한 기술적 관점을 제공하고, 사용자의 성향에 따라 추천 민감도를 조절할 수 있게 한다 |
| 타겟 유저 원칙 유지 | 초보자 대상 원칙은 유지 — **기본 UX는 프리셋 3버튼**이며, 자유 설정은 기본적으로 접혀 있는 "고급 설정" 패널 안에만 존재한다. 고급 설정을 열지 않는 사용자는 그 존재를 의식할 필요가 없다 |

## 2. 선행 작업 — 데이터 수집 기간 변경 (6→12개월)

52주 신고가/신저가 계산에는 최근 252거래일(약 12개월)이 필요하다.

- `scripts/collect_data.py`: `MONTHS_BACK = 6 → 12`, `MIN_TRADING_DAYS = 90 → 200`
- JSON 스키마는 변경 없음 (series 배열이 길어질 뿐). 파일 용량 약 2배 — GitHub Pages 배포에 문제없는 수준
- **⚠️ 파급효과 처리 필수**: v5의 `chart.sixMonth`가 "수집된 전체 기간"으로 정의되어 있어,
  12개월 수집 시 6개월 차트가 12개월 차트가 되어버린다 → `chart.sixMonth = 최근 126거래일`로 정의 변경
- 지표·시뮬레이션의 기존 계산 창(RSI 14, MACD, 3개월 앵커 63거래일 등)은 "최근 N거래일" 기준이므로 영향 없음 — 단 회귀 테스트로 확인
- 재수집 후 `generatedAt` 갱신 → 리서치 서브에이전트의 `basedOnDataOf` 정합성 경고가 정상 작동하는지 확인

## 3. 범위

### Must have
1. **볼린저밴드 필터** (화면1) — SMA20 ± 2×표준편차(20일)
   - 옵션 2종 중 택1: "하단 밴드 근접" (종가 ≤ 하단밴드 × 1.02) / "상단 밴드 돌파" (종가 ≥ 상단밴드)
2. **52주 신고가/신저가 필터** (화면1) — 최근 252거래일 기준
   - 옵션 2종 중 택1: "52주 신고가 근접" (신고가 대비 −5% 이내) / "52주 신저가 근접" (신저가 대비 +5% 이내)
3. **스토캐스틱 필터** (화면1) — Slow Stochastic (14, 3, 3)
   - 옵션 2종 중 택1: "과매도 구간" (%K ≤ 20) / "과매수 구간" (%K ≥ 80)
4. **ATR 변동성 필터** (화면1) — ATR(14, Wilder 평활)을 종가로 나눈 **ATR%** 기준, 유니버스 내 상대 백분위
   - 옵션 2종 중 택1: "저변동성" (ATR% 하위 30%) / "고변동성" (ATR% 상위 30%)
   - 절대 임계가 아닌 백분위인 이유: 시장 국면에 따라 전체 변동성 수준이 달라지므로 상대 기준이 안정적
5. **OBV 거래량 흐름 필터** (화면1) — On-Balance Volume
   - 옵션 2종 중 택1: "OBV 상승 추세 (매집 신호)" (OBV > OBV의 SMA20) / "OBV 하락 추세 (분산 신호)" (OBV < OBV의 SMA20)
6. **필터 원칙 유지**: 신규 필터 5종 모두 첫 진입 시 꺼짐, 기존 4종과 자유 조합 가능, localStorage 저장
7. **추천 프리셋 3종** (화면2 상단에서 선택):

| 파라미터 | 보수형 | 기본형 (=v5 현행) | 공격형 |
|---|---|---|---|
| RSI 하한 | ≥ 55 | ≥ 50 | ≥ 45 |
| 골든크로스 창 | 3거래일 | 5거래일 | 10거래일 |
| 고득점 편입 임계 | 80점 | 70점 | 60점 |
| 완화 폴백 | 동일 로직 유지 (창 2배 → RSI만) |||

   - 기본값은 "기본형" — 프리셋 미선택 사용자는 v5와 완전히 동일한 결과를 본다
   - 2단계 배점(이격도 60/거래량 30/섹터 10)은 프리셋과 무관하게 고정 (배점 커스텀은 확장 범위)
   - 선택한 프리셋은 localStorage에 저장, 추천 결과 상단에 현재 프리셋 명시
8. **`recommend.js` 설정 객체화** — 하드코딩 상수를 `presets.js`의 설정 객체로 추출.
   기본형 설정으로 실행 시 v5와 결과가 바이트 단위로 동일해야 함 (회귀 기준)
9. **고급 설정 (자유 파라미터 조정)** — 프리셋 세그먼트 아래 접이식 패널 (기본 접힘):
   - 조정 가능 파라미터는 프리셋이 다루는 3종과 동일: RSI 하한 / 골든크로스 창 / 고득점 편입 임계
   - 허용 범위 강제: RSI 하한 30~70, 골든크로스 창 1~20거래일, 편입 임계 50~95점 (범위 밖 입력 불가)
   - 파라미터를 하나라도 프리셋 값과 다르게 바꾸면 상태가 **"사용자 설정(custom)"**으로 전환되고

   - "기본형으로 초기화" 버튼 제공
   - 극단 설정으로 결과가 0개가 되는 경우는 기존 `insufficientSignal` 배너가 그대로 처리 (별도 방어 불필요)

## 4. 핵심 기능 상세

### 4.1 신규 지표 계산 (`lib/indicators.js` 추가)
| 함수 | 정의 |
|---|---|
| `bollingerBands(series, 20, 2)` | 중심선 SMA20, 상단 = SMA20 + 2σ, 하단 = SMA20 − 2σ (σ = 20일 종가 표본표준편차) |
| `week52HighLow(series)` | 최근 252거래일의 최고 고가·최저 저가. 데이터가 252일 미만이면 **해당 종목은 이 필터에서 제외** (전체 기간으로 대충 계산하지 않음 — "52주"의 의미 왜곡 방지) |
| `stochastic(series, 14, 3, 3)` | Fast %K = (종가 − 14일 최저 저가) / (14일 최고 고가 − 14일 최저 저가) × 100, Slow %K = Fast %K의 SMA3, %D = Slow %K의 SMA3. 분모 0(14일간 고저 동일)이면 해당일 값 null 처리 |
| `atr(series, 14)` | True Range = max(고−저, \|고−전일종가\|, \|저−전일종가\|), ATR = TR의 Wilder 평활(14). 필터용 ATR% = ATR / 종가 × 100. 백분위는 필터 적용 시점에 유니버스 전체(dataSufficient 종목)의 ATR% 분포에서 계산 |
| `obv(series)` | 종가 상승일 +거래량, 하락일 −거래량, 보합 0의 누적합. 필터 판정은 OBV와 OBV의 SMA20 비교 |

### 4.2 화면1 필터 UI — 그룹핑 재편
- 필터가 총 9종(기존 4 + 신규 5)이 되므로, 나열식 유지 시 화면1이 과밀해진다 → **성격별 그룹으로 재편**:

| 그룹 | 포함 필터 |
|---|---|
| 모멘텀 | 과열/과매도(RSI, 기존), 스토캐스틱 🆕 |
| 추세·가격 위치 | 이격도(기존), 볼린저밴드 🆕, 52주 신고가/신저가 🆕 |
| 거래량 | 거래량 증가(기존), OBV 🆕 |
| 변동성 | ATR 🆕 |
| 시장 구조 | 주도주 섹터(기존) |

- 각 그룹은 접이식 섹션, 활성 필터가 있는 그룹은 헤더에 활성 개수 배지 표시
- 개별 필터는 옵션 선택형(라디오/셀렉트)이며 꺼짐이 기본 — **기존 필터의 동작·판정 로직은 재편 대상 아님** (UI 배치만 변경)
- 252일 미만 종목이 52주 필터로 걸러질 때, 필터 영역에 "데이터 12개월 미만 종목 N개 제외됨" 안내 표시
- 유사 개념 필터 동시 적용(예: RSI 과매도 + 스토캐스틱 과매수)은 막지 않되, 결과 0개 시 기존 빈 결과 안내가 처리

### 4.3 추천 프리셋 + 고급 설정 (화면2)
- 화면2 상단에 프리셋 선택 세그먼트 (보수형/기본형/공격형 + 상태 표시용 "사용자 설정")
- 세그먼트 아래 접이식 **"고급 설정"** 패널 (기본 접힘):
  - RSI 하한 / 골든크로스 창 / 편입 임계 3개 입력 (숫자 입력 또는 슬라이더, 허용 범위 강제)
  - 각 입력 옆에 현재 프리셋 대비 변경 여부 표시 (예: 기본형 50 → 현재 55)
  - "기본형으로 초기화" 버튼
- 프리셋 변경·파라미터 조정 즉시 추천 재계산 (클라이언트 계산이므로 통신 없음)
- 완화 폴백 발동·"신호 부족" 배너는 **현재 적용 중인 설정 기준**으로 판정하고, 배너 문구에
  프리셋명 또는 "사용자 설정"을 포함 (예: "사용자 설정 기준 매수 신호가 충분치 않습니다")

### 4.4 리서치 서브에이전트와의 정합성 (v6 연계)
- **서브에이전트(US-1 CLI)는 항상 "기본형" 고정** — 리서치 풀 선정 기준이 사용자마다 달라지는 것을 방지
- 화면2에서 기본형이 아닌 프리셋 또는 사용자 설정 적용 중일 때, 리서치 섹션이 있는 카드에
  "리서치 풀은 기본형 기준으로 선정되었습니다" 보조 문구 표시
- `research.json` 스키마 변경 없음

4. 필터 임계 상수는 모두 `lib/constants.js`에서 관리: 52주 근접 ±5%, 볼린저 하단 근접 1.02, 스토캐스틱 20/80, ATR 백분위 30%/70%, OBV SMA 창 20
