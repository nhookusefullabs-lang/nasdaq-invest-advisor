# prd-v8-dual-mode.md — 이중 모드 & 펀더멘털 허들 (Ralph 호환 태스크 문서)

> 원본 설계: `PRD_Nasdaq8.md` (v8). 이 문서는 Ralph 루프가 한 이터레이션에 스토리 하나씩
> 구현→검증→커밋하도록 변환한 실행용 문서다.
> 루프 규칙: 매 이터레이션마다 **미완료 스토리 중 가장 위의 것 하나만** 골라 구현하고,
> 승인 기준을 전부 통과하면 체크박스를 채우고 커밋한 뒤 이터레이션을 종료한다.

## 프로젝트 컨텍스트 (매 이터레이션 필독)

- 기존 시스템: React+Vite+Tailwind 정적 SPA (백엔드 없음), GitHub Pages 배포, v7(as-built)까지 운영 중
- 데이터 패턴: `public/data/` 정적 스냅샷 (`nasdaq100.json`, `research.json`). 이번 작업으로
  `fundamentals.json`이 추가되고 nasdaq100.json 수집 기간이 2년으로 확대된다
- 기존 추천 로직: `src/lib/recommend.js` (추세추종 2단계 + 프리셋/고급설정) — **이번 버전에서 로직 변경 금지**
- 테스트: vitest (**기존 테스트 전체 통과 상태 유지 필수 — 깨뜨리면 해당 이터레이션 실패**)
- 절대 원칙:
  - 기존 화면·로직의 **동작 변경 금지 (추가만 허용)**. 추세추종 단일 모드 선택 시 화면2는 v7과 기능적으로 동일해야 함
  - `fundamentals.json`·`research.json`이 없어도 앱은 완전 정상 동작 (graceful degradation)
  - 모든 신규 판정(트렌드 템플릿·허들·배지)은 근거를 화면에서 확인 가능 (블랙박스 금지)
  - 새 npm 의존성 추가 금지 (필요 시 이터레이션 중단하고 사유 기록)
  - 배점·임계값 상수는 `src/lib/constants/v8.js` 단일 모듈에만 정의 (v9 백테스트 조정 대비)
  - 실데이터 수집 실행(yfinance)은 루프가 하지 않는다 — 스크립트·검증기·픽스처까지만 만들고 실행은 운영자 몫
- 품질 게이트 (모든 스토리 공통): `npm run test` 전체 통과 + `npm run build` 성공

### US-3. 공유 지표 계층 + 상수 모듈

### US-6. 컨센서스 랭킹
- [ ] **사용자로서, 두 관점이 동시에 긍정하는 종목을 최상위로 보고 싶다.**
