# /research — 나스닥100 리서치 서브에이전트

이 커맨드는 `PRD_Nasdaq6.md`(v6 설계)와 `prd-research-agent.md`(US-1~US-7)에 따라
추천 종목에 대한 정성적 리서치를 수행해 `public/data/research.json` 스냅샷을 생성하는
운영 절차를 정의한다. 실행 주체는 Claude Code(로컬, 수동 실행)이며, 데이터 수집
(`collect_data.py`)과 같은 날(토요일) 직후 실행하는 것을 권장한다.

## 절차 (6단계)

### ① US-1 CLI로 추천 풀 제시
`node scripts/research/select-pool.mjs`를 실행해 현재 `nasdaq100.json` 기준 추천 풀
(상위 10개 + 고득점 특별 편입, 티커/이름/점수/신호통과여부/사유)을 운영자에게 표로 제시한다.

### ② 운영자 대화형 선택 + 관심 종목 입력(검증)
운영자가 ①의 풀에서 리서치할 종목을 선택한다 (전체 선택 가능). 추천 풀과 무관하게
직접 리서치를 원하는 티커가 있다면 `node scripts/research/select-pool.mjs <티커...>`로
나스닥100 유니버스 소속 여부와 `dataSufficient` 여부를 사전 검증한 뒤 대상에 추가한다.
유니버스 밖 티커나 데이터 부족 티커는 사유와 함께 거부하고 대상에서 제외한다.
**1회 세션 리서치 대상은 최대 15종목**으로 제한한다.

### ③ 종목별 리서치 — 계층 A (자동 조회)
확정된 대상 종목마다 아래 우선순위로 무료 공개 소스를 조회한다. **종목당 검색 3~5회
상한**을 지켜 토큰 예산을 통제한다.

1. **SEC EDGAR (13F 보고서)** — 기관투자자 분기별 보유 현황 → `institutionalActivity`
2. **Dataroma** — 슈퍼인베스터 13F 집계 (EDGAR와 교차 확인 권장)
3. **Macrotrends** — 장기 재무 추이 (매출/마진/PER/부채) → 촉매·리스크 근거 보강
4. **TipRanks** — 애널리스트 컨센서스·목표주가 → `analystView` (공개 범위만)
5. **Zacks Investment Research** — Zacks Rank·실적 전망 수정 방향 → `analystView`
6. **The Motley Fool** — 종목 분석 기사 (단일 근거로 쓰지 않고 교차 확인)
7. **초이스스탁US** — 한국어 재무 요약 (적정주가 등 자체 산출값은 참고 수준으로만)
8. **일반 웹검색(뉴스)** — 위 소스가 커버 못 하는 최신 촉매·리스크 보완 (원출처 우선)

소스 하나가 로그인 벽·차단·구조 변경으로 실패해도 **종목 전체를 skip하지 않고 다음
소스로 진행**한다. 13F·Dataroma 기반 정보는 시차가 크므로(분기 종료 후 45일) 최근
뉴스와 시점이 어긋나면 summary에서 기준 시점을 명시한다.

### ④ 계층 B 확인 요청 (회원 콘텐츠, human-in-the-loop)
아래 소스는 자동 접근이 불가능하거나 이용약관상 부적절하므로, 운영자에게 확인을 요청한다.

- **Value Investors Club (VIC)** — 게스트 열람(약 90일 지연) 범위는 계층 A처럼 시도,
  회원 전용 글은 운영자 확인 요청
- **Seeking Alpha** — 강한 페이월·봇 차단으로 자동 접근 불가, 운영자 확인 요청 전용
- **IBD50** — 리스트 자체가 구독 콘텐츠, 대상 종목의 IBD50 편입 여부를 운영자가 확인

에이전트는 종목별로 "위 소스 중 확인 가능한 것이 있으면 등급/논거/편입 여부를 요약해
알려달라"고 요청한다. **운영자 입력이 없으면 계층 A 결과만으로 해당 종목 리서치를
완결한다** — 계층 B는 필수가 아니라 보강용이다. 운영자가 제공한 내용은
`sources[]`에 `operatorProvided: true`와 열람일·URL(가능한 경우)을 함께 기록한다.

### ⑤ US-2 검증기로 원자적 저장
수집한 항목을 `research.json` 스키마(버전 1, `PRD_Nasdaq6.md` §4.2)에 맞춰 조립한 뒤
`scripts/research/validate-research.mjs`(또는 `atomicWriteResearch` 헬퍼)로 검증하고
저장한다. 검증 실패 시 기존 `research.json`은 훼손되지 않는다 (임시 파일에 먼저 쓰고
통과 시에만 rename). 출처가 하나도 확보되지 않은 종목은 `items`가 아니라 `skipped`에
사유와 함께 기록한다.

### ⑥ 결과 요약 보고
저장 완료 후 운영자에게 다음을 요약 보고한다: 리서치 완료 종목 수, skip된 종목과 사유,
센티먼트 분포, 계층 B 입력을 받은 종목 목록. 이후 커밋·배포는 운영자가 수동으로
검토한 뒤 진행한다 (`PRD_Nasdaq6.md` §4.1 [5]).

## 규칙 (반드시 준수)

1. **매수/매도 권유 표현 금지** — "사야 한다", "유망하다" 등 투자 권유성 표현을
   `summary`/`catalysts`/`risks`/`analystView`에 사용하지 않는다. 사실 요약만 담는다.
2. **13F 기반 정보는 기준 분기를 명시**한다 (예: "1분기 기준 ○○ 펀드 신규 편입").
3. **애널리스트 등급(TipRanks/Zacks)은 수집 시점 기준임을 `analystView`에 명시**한다.
4. **로그인 계정 정보(아이디/비밀번호)를 절대 받지 않으며, 로그인 벽 우회 자동화를
   시도하지 않는다** (이용약관 준수).
5. **운영자가 제공한 내용은 `operatorProvided: true`로 기록**하고 열람일을 남긴다.
6. **소스 하나의 접근 실패는 종목 실패가 아니다** — 대체 소스로 계속 진행한다.
7. **출처가 1개도 없는 종목은 `items`가 아니라 `skipped`로 처리**한다.
8. **유료·회원 콘텐츠 원문을 복사하지 않는다** — 재서술 요약만 담고 인용은 출처 링크로
   대체한다.
9. **1회 세션 리서치 대상은 최대 15종목**을 넘지 않는다.

## Out of Scope

- 배포된 웹 화면에서 리서치를 요청하는 기능 (백엔드 없음 — 대상 선정은 이 커맨드
  실행 시점의 대화형 선택으로만 가능)
- 나스닥100 유니버스 밖 종목 리서치
- 리서치 이력 누적 보관 (스냅샷 1개만 유지, 덮어쓰기)
- 자동 스케줄 실행 (수동 실행만 지원)
